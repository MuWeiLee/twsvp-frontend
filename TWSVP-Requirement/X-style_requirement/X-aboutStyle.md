# TWSVP Mobile Web App UI 规范

## 一、产品形态定位
「一个长时间打开、单手操作、信息密集的投资 App」

## 二、整体视觉风格

### 1. 页面底色
- 极浅灰（接近 iOS Notes / Threads 背景）
- 不是白色、不是深色

### 2. 文字体系（黑字为主）
- 主文案：深灰黑（非纯黑）
- 次要信息：中灰
- Meta：浅灰

### 3. 去 AI 化强规则
**禁止使用：**
- 渐变
- 发光
- 玻璃拟态
- “AI 蓝 / 紫”
- 聊天气泡
- 动画思考状态

## 三、移动 App 布局结构

### 标准页面结构
```
┌──────── Top Bar ────────┐
│ 标题 / 搜索 / 操作     │
├────────────────────────┤
│ Filter / Tabs（可选）  │
├────────────────────────┤
│ Content List           │ ← 主滚动区
│  ├─ Card               │
│  ├─ Card               │
│  └─ Card               │
└──────── Bottom Nav ─────┘
```

### 关键规则
- Top Bar 固定
- Bottom Nav 固定
- 中间区域独立滚动（像 App）

## 四、Top Bar

### 样式
- 高度：48–52px
- 背景：白或浅灰
- 底部 1px 分隔线
- 很浅的阴影

### 内容
- 左：页面标题 / 返回
- 中：可选 Search（或点击进入搜索页）
- 右：Icon Button（收藏 / 分享 / 设置）

## 五、Bottom Navigation

### 规则
- 固定在底部
- 高度：56px

### 样式
- 图标 + 文本
- 当前项：
  - 文本加深
  - 图标填充
  - 不要彩色背景

## 六、核心组件规范（全部 App 化）

### 1. Feed Card（移动端核心）

#### 卡片外观
- 白底
- 1px 边框
- 圆角 8px
- 无阴影

#### 卡片内边距
- 上下：12px
- 左右：12px

#### 信息结构（紧凑）
- [ 标的代码  名称        状态Badge（未结束 / 已结束） ]
- [ Meta：标的 / 看法 / 时效 ]
- [ 摘要文本（3 行） ]
- [ 点赞 / 时间]

#### 原则
- 一屏至少能看到 2–3 张卡片
- 文本截断，不展开即克制

### 2. 状态 Badge（极简）
- 高度：20px 左右
- 边框 + 文字
- 不使用实色块

### 3. Search 页（独立页，不是弹窗）
- 顶部大搜索框
- 下方包含：
  - 最近搜索
  - 热门标的
  - 快速入口

**非常重要：** 搜索页 ≠ AI 对话页

### 4. Filter / Tabs（拇指友好）
- 横向滚动
- 文本 Tab
- 选中态仅 underline

## 七、详情页 / Wiki（文档感）

### Wiki / 说明页
- 白底
- 黑字
- 类 Notion / Vue 文档
- 术语 = 粗体 + 一句话解释

### Modal / Bottom Sheet
- 从底部弹出
- 圆角大一点（12–16px）
- 顶部拖拽条

## 八、交互反馈（像原生 App）

### 点击反馈
- iOS 风格：背景轻微变深
- 无缩放、无发光

### Loading
- Skeleton
- 占位块
