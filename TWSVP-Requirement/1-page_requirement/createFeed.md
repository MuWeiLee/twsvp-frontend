0. 基本信息

页面英文名：CreateFeed

页面层级：#3 表单流程页

访问前提：已登录

对应核心数据对象：Feed

1. 页面目标（Goal）

让用户快速发布一条观点 Feed

保证观点具备：

明确标的

明确看法

明确时效

内容表达保持完全自由

2. 页面整体结构（Layout）
顶部栏

| 发布观点 | 关闭 |

置顶内容层（观点判断信息）

标的选择（Stock / Sector）

看法选择：

看多 / 看空 / 中性

时效选择：

短期（5–20 天）

中期（20–60 天）

长期（60–180 天）

⚠️ 上述字段 创建后不可编辑

内容层（Content Area）

观点内容

类型：多行文本（textarea）

字段：content

占位提示（示例）：
```
Placeholder:
  用一句话说明你的核心判断
  你的判断依据
  关于观点的风险提示...
```
校验规则（建议）：

必填

最小长度：20 字

最大长度：1000 字

底部操作区

发布按钮（校验通过后可点）

3. 交互流程

用户点击「发布」

填写标的 / 看法 / 时效

输入观点内容

点击发布

系统创建 Feed

跳转 FeedCard 或返回 FeedFlow

4. 验收标准

 未填写 content 无法发布

 Feed 创建成功后状态为 active

 expired_at 根据 horizon 自动生成

- 发布按钮（主 CTA）

#### 底部 Tab
- 发布（高亮）

- target_id
- direction
- horizon
- content

### 5. 状态与规则

- 校验未通过不可发布
- 发布成功立即进入 FeedFlow

- 必填字段校验有效

## FeedCard（观点详情）

### 0. 基本信息

- **页面名称**：观点详情
- **页面英文名**：FeedCard
- **页面层级**：#3 内容详情
- **访问前提**：已登录
- **入口来源**：FeedFlow / StockFeed / Search
- **页面出口**：返回上一级
- **是否在底部 Tab 中**：否
- **对应核心数据对象**：Feed

- 完整展示单条 Feed
